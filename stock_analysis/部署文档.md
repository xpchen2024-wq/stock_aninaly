## 1. 部署架构概览

```
                    ┌──────────────────────────────────────────┐
                    │              Nginx / Traefik              │
                    │           (TLS + 反向代理)                │
                    └──────────────┬───────────────────────────┘
                                   │
                    ┌──────────────▼───────────────────────────┐
                    │           FastAPI (4 workers)             │
                    │           REST + WebSocket API            │
                    └──┬────────┬────────┬─────────────────────┘
                       │        │        │
          ┌────────────▼──┐  ┌──▼───────▼──┐  ┌──────────────────┐
          │  PostgreSQL   │  │   Redis     │  │  Celery Worker   │
          │  (业务数据)    │  │  (缓存)     │  │  (定时任务)       │
          └───────────────┘  └─────────────┘  └──────────────────┘
                                                       │
                                              ┌────────▼─────────┐
                                              │  Celery Beat     │
                                              │  (任务调度)       │
                                              └──────────────────┘
```

### 服务组件

| 组件 | 端口 | 说明 |
|------|:----:|------|
| FastAPI | 8000 | REST API + WebSocket |
| PostgreSQL | 5432 | 业务数据库 |
| Redis | 6379 | 缓存 + 消息队列 |

| Celery Beat | — | 定时任务调度 |

## 3. Docker Compose 部署（推荐）

### 3.1 克隆项目

```bash
cd /Users/cxp/myAIcoding/stock_analysis
```

### 3.2 配置环境变量

```bash
cp .env.example .env
vim .env
```

**必须修改的配置项**：

```bash
# JWT 密钥（至少 32 字符随机字符串）
JWT_SECRET_KEY=your-random-secret-key-at-least-32-characters-long

# AES 加密密钥（至少 32 字节）
ENCRYPTION_KEY=your-32-byte-encryption-key-here!!!!

# AI 模型 API Key（至少配一个）
OPENAI_API_KEY=sk-your-actual-openai-key
# 或
DEEPSEEK_API_KEY=sk-your-deepseek-key
```

生成安全密钥的方法：

```bash
# JWT Secret
openssl rand -hex 32

# Encryption Key (32 bytes)
openssl rand -base64 32
```

### 3.3 启动服务

```bash
# 构建并启动所有核心服务
docker compose up -d

# 包含监控组件启动
docker compose --profile monitoring up -d

# 查看启动状态
docker compose ps
```

### 3.4 验证部署

```bash
# 健康检查
curl http://localhost:8000/health
# 预期: {"status":"healthy","version":"1.4.0"}

# 查看API文档
open http://localhost:8000/docs

# 检查数据库
docker exec stock_postgres psql -U stock_user -d stock_analysis -c "\dt"
# 预期: 列出 13 张表

# 检查 Redis
docker exec stock_redis redis-cli ping
# 预期: PONG
```

### 3.5 停止/重启

```bash
# 停止所有服务
docker compose down

# 停止并删除数据卷（⚠️ 会丢失数据）
docker compose down -v

# 重启单个服务
docker compose restart api
```

## 4. 本地开发部署

### 4.1 安装系统依赖

**macOS**：

```bash
# 安装 TA-Lib C 库
brew install ta-lib

# 安装 PostgreSQL 和 Redis
brew install postgresql@15 redis
brew services start postgresql@15
brew services start redis
```

**Ubuntu/Debian**：

```bash
sudo apt update
sudo apt install -y build-essential wget
sudo apt install -y postgresql postgresql-contrib redis-server

# TA-Lib
wget http://prdownloads.sourceforge.net/ta-lib/ta-lib-0.4.0-src.tar.gz
tar -xzf ta-lib-0.4.0-src.tar.gz
cd ta-lib && ./configure --prefix=/usr && make && sudo make install
cd .. && rm -rf ta-lib*
```

### 4.2 创建数据库

```bash
sudo -u postgres psql <<EOF
CREATE USER stock_user WITH PASSWORD 'stock_pass';
CREATE DATABASE stock_analysis OWNER stock_user;
GRANT ALL PRIVILEGES ON DATABASE stock_analysis TO stock_user;
EOF
```

### 4.3 安装 Python 依赖

```bash
cd /Users/cxp/myAIcoding/stock_analysis

# 建议使用虚拟环境
python3.11 -m venv venv
source venv/bin activate

pip install -r requirements.txt
```

### 4.4 初始化数据库

```bash
psql -U stock_user -d stock_analysis -h localhost -f db/init_db.sql
psql -U stock_user -d stock_analysis -h localhost -f db/seed_data.sql
```

### 4.5 启动服务

```bash
# 终端 1: 启动 API
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 终端 2: 启动 Celery Worker
celery -A app.celery_app worker --loglevel=info

# 终端 3: 启动 Celery Beat（定时任务）
celery -A app.celery_app beat --loglevel=info
```

## 6. 数据库初始化

### 6.1 自动初始化（Docker）

Docker Compose 会在 PostgreSQL 首次启动时自动执行 `db/init_db.sql` 和 `db/seed_data.sql`。

### 6.2 手动初始化

```bash
# 1. 创建数据库和用户
sudo -u postgres psql
CREATE USER stock_user WITH PASSWORD 'stock_pass';
CREATE DATABASE stock_analysis OWNER stock_user;
\q

# 2. 执行建表脚本
psql -U stock_user -d stock_analysis -f db/init_db.sql

# 3. 执行种子数据
psql -U stock_user -d stock_analysis -f db/seed_data.sql
```

### 6.3 数据库表结构

系统共创建 **13 张表**：

| 表名 | 说明 |
|------|------|
| `users` | 用户认证 |
| `watchlist_groups` | 关注列表分组 |
| `watchlist_items` | 关注列表项 |
| `kline_cache` | K线缓存（关注列表 90 天） |
| `stock_fundamentals` | 股票基本面 |
| `model_configs` | AI 模型配置 |
| `agent_model_bindings` | Agent-场景-模型绑定 |
| `news_articles` | 新闻文章 |
| `hot_topics` | 热点话题 |
| `research_reports` | 研报 |
| `serenity_analyses` | Serenity 分析结果 |
| `trend_analyses` | 趋势分析结果 |
| `agent_decisions` | Agent 决策记录 |
| `kols` | 大V信息 |
| `kol_opinions` | 大V观点 |
| `kol_consensus` | 大V共识 |
| `analysis_conclusions` | AI 结论统一存储 |
| `audit_logs` | 审计日志 |

### 6.4 数据清理

系统内置定时清理任务（每日 02:00 执行）：

- 新闻 > 72 小时 → 标记归档
- 大V观点 > 30 天 → 删除
- K线缓存 > 90 天 → 删除
- 过期热点 → 删除
- 审计日志 > 90 天 → 删除

手动执行清理：

```bash
docker exec stock_postgres psql -U stock_user -d stock_analysis \
  -c "SELECT cleanup_expired_data();"
```

## 7. AI 模型配置

### 7.1 通过 API 配置模型

```bash
# 登录获取 Token
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}' | jq -r .access_token)

# 添加 OpenAI 模型
curl -X POST http://localhost:8000/api/v1/models \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "GPT-4o",
    "platform": "openai",
    "base_url": "https://api.openai.com/v1",
    "api_key": "sk-your-key",
    "model_name": "gpt-4o",
    "max_tokens": 4096,
    "temperature": 0.7,
    "is_default": true
  }'

# 测试连接
curl -X POST http://localhost:8000/api/v1/models/{id}/test \
  -H "Authorization: Bearer $TOKEN"
```

### 7.2 支持的平台

| 平台 | Base URL | 代表模型 |
|------|----------|---------|
| OpenAI | https://api.openai.com/v1 | gpt-4o, gpt-4o-mini, o1 |
| DeepSeek | https://api.deepseek.com/v1 | deepseek-chat, deepseek-reasoner |
| OpenRouter | https://openrouter.ai/api/v1 | claude, gemini, llama |
| Custom | 用户自定义 | 任意 OpenAI 兼容模型 |

### 7.3 Agent-场景绑定

不同 Agent 角色和分析场景可绑定不同模型：

```bash
# 绑定趋势分析场景到 DeepSeek
curl -X POST http://localhost:8000/api/v1/models/bindings \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model_config_id": "deepseek-config-id",
    "scene": "trend_analysis"
  }'

# 绑定 Serenity 深度分析到 GPT-4o
curl -X POST http://localhost:8000/api/v1/models/bindings \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model_config_id": "gpt4o-config-id",
    "scene": "serenity"
  }'
```

## 8. 定时任务配置

### 8.1 任务列表

| 任务 | 队列 | 频率 | 说明 |
|------|------|------|------|
| `crawl_news` | crawler | 15 min | 新闻采集 |
| `compute_hot_topics` | analysis | 15 min | 热点计算 |
| `crawl_kol_opinions` | crawler | 30 min | 大V观点采集 |
| `generate_kol_consensus` | analysis | 每日 18:00 | 大V共识生成 |
| `cleanup_expired_data` | maintenance | 每日 02:00 | 过期数据清理 |

```bash

# 手动触发任务

celery -A app.celery_app call crawl_kol_opinions
celery -A app.celery_app call generate_kol_consensus
celery -A app.celery_app call cleanup_expired_data
```

### 8.3 查看任务状态

```bash
# 查看活跃任务
celery -A app.celery_app inspect active

# 查看已注册任务
celery -A app.celery_app inspect registered

```

## 10. 常见问题

### Q1: TA-Lib 安装失败

**问题**：`pip install TA-Lib` 报错 `fatal error: ta-lib/ta_libc.h: No such file`

**解决**：需要先安装 TA-Lib C 库

```bash
# macOS
brew install ta-lib

# Linux
wget http://prdownloads.sourceforge.net/ta-lib/ta-lib-0.4.0-src.tar.gz
tar -xzf ta-lib-0.4.0-src.tar.gz
cd ta-lib
./configure --prefix=/usr
make
sudo make install
```

### Q2: 数据库连接失败

**问题**：`connection to server at "localhost" (127.0.0.1), port 5432 failed`

**解决**：

1. 确认 PostgreSQL 正在运行：`docker compose ps postgres`
2. 确认 `.env` 中 `DATABASE_URL` 配置正确
3. Docker 部署时使用服务名：`postgresql+asyncpg://stock_user:stock_pass@postgres:5432/stock_analysis`

### Q3: AI 模型调用失败

**问题**：LLM 调用返回 `No model configured`

**解决**：

1. 确认已在 `.env` 中配置 API Key
2. 通过 API 添加模型配置（见第 7 节）
3. 测试连接：`POST /api/v1/models/{id}/test`

### Q4: Celery 任务不执行

**问题**：定时任务没有触发

**解决**：

1. 确认 Celery Beat 正在运行：`docker compose ps celery_beat`
2. 检查 Beat 日志：`docker compose logs celery_beat`
3. 手动触发测试：`celery -A app.celery_app call crawl_news`

### Q5: 大V观点采集为空

**问题**：`GET /api/v1/kol/opinions` 返回空列表

**说明**：大V观点采集需要对接抖音/微博平台 API 或爬虫，当前 `tasks.py` 中的采集函数为占位实现。生产环境需：

1. 接入抖音开放平台 API
2. 接入微博开放平台 API
3. 或使用第三方数据采集服务

种子数据中已包含 20 名大V信息（`db/seed_data.sql`），可通过 `POST /api/v1/kol/kols` 手动管理。
